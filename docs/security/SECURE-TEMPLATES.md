# Secure HTML Templates

This document describes the secure HTML template pattern used in Storm Scout to prevent Cross-Site Scripting (XSS) vulnerabilities.

## Overview

Storm Scout uses a **tagged template literal** approach to automatically escape dynamic content inserted into HTML. This provides XSS protection by default while maintaining readable template syntax.

## Functions

### `html` - Safe HTML Template Tag

The `html` tagged template automatically escapes all interpolated values:

```javascript
// Values are automatically escaped
const userInput = '<script>alert("xss")</script>';
container.innerHTML = html`<div>${userInput}</div>`;
// Result: <div>&lt;script&gt;alert(&quot;xss&quot;)&lt;/script&gt;</div>
```

### `escapeHtml(value)` - Manual Escaping

For cases where you need to escape a value outside of a template:

```javascript
const safe = escapeHtml(unsafeValue);
```

### `raw(html)` - Trusted HTML Marker

For HTML strings that you know are safe (constructed by your code, not user input):

```javascript
const badge = '<span class="badge bg-success">Active</span>';
container.innerHTML = html`<div>${data.name} ${raw(badge)}</div>`;
```

⚠️ **WARNING**: Never use `raw()` with user input or external data!

## Usage Examples

### Basic Usage

```javascript
// Before (vulnerable)
container.innerHTML = `<div>${data.title}</div>`;

// After (secure)
container.innerHTML = html`<div>${data.title}</div>`;
```

### With Trusted HTML

```javascript
// Badge HTML is constructed by our code, so it's trusted
function renderSite(site) {
    const severityBadge = `<span class="badge ${getSeverityBadge(site.severity)}">${escapeHtml(site.severity)}</span>`;
    
    container.innerHTML = html`
        <div class="card">
            <h3>${site.name}</h3>
            <p>${site.description}</p>
            ${raw(severityBadge)}
        </div>
    `;
}
```

### Complex Templates

```javascript
function renderAdvisoryRow(adv) {
    return html`
        <tr>
            <td><strong>${adv.site_code}</strong></td>
            <td>${adv.site_name}</td>
            <td>${adv.city}</td>
            <td>${adv.state}</td>
            <td>${adv.advisory_type}</td>
            <td>${raw(getSeverityBadge(adv.severity))}</td>
            <td>${adv.source}</td>
        </tr>
    `;
}

tbody.innerHTML = advisories.map(renderAdvisoryRow).join('');
```

## Migration Guide

### Step 1: Identify innerHTML Assignments

Find all places where innerHTML is set with template literals:

```javascript
element.innerHTML = `...${dynamicValue}...`;
```

### Step 2: Add html Tag

Add the `html` prefix to the template:

```javascript
element.innerHTML = html`...${dynamicValue}...`;
```

### Step 3: Handle Trusted HTML

If the template includes HTML generated by your code (like badges), wrap it with `raw()`:

```javascript
// Before
element.innerHTML = `<div>${getBadgeHtml(status)}</div>`;

// After
element.innerHTML = html`<div>${raw(getBadgeHtml(status))}</div>`;
```

## Security Guidelines

### DO ✅

- Use `html` tagged templates for all innerHTML assignments
- Use `escapeHtml()` inside functions that generate trusted HTML
- Review `raw()` usage carefully in code reviews

### DON'T ❌

- Never use `raw()` with user input or external API data
- Don't use plain template literals with innerHTML
- Don't disable escaping for "trusted" data sources (databases can be compromised)

## Code Review Checklist

When reviewing code that uses innerHTML:

1. [ ] Is the `html` tag used?
2. [ ] Are all `raw()` calls using internally-generated HTML only?
3. [ ] Is external data (API responses, user input) always escaped?
