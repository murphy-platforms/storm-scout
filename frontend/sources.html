Data Sources

Information about where Storm Scout gets its data

Primary Weather Data Source
NOAA / National Weather Service

The National Oceanic and Atmospheric Administration (NOAA) and National Weather Service (NWS) provide real-time weather alerts, warnings, and advisories for the United States. Storm Scout uses the official NOAA Weather API to fetch active alerts.

  • Update Frequency: Every 15 minutes (configurable)
  • Coverage: All 50 US states and territories
  • Data Types: Watches, Warnings, Advisories, Statements
  • Reliability: Official government source - highly reliable
Visit NOAA Weather API Documentation
Emergency Management Data
State & Local Emergency Management

Storm Scout is designed to integrate with state and local emergency management feeds to capture:

  • State of Emergency declarations
  • Evacuation orders
  • County/city emergency notices
  • FEMA disaster declarations

Note: This feature is a placeholder for future implementation. Specific state/local data sources will be added based on availability and API access.

Site Location Data
Testing Center Locations

Storm Scout monitors 220 testing center locations across the United States, including:

  • All 50 US states
  • US territories (Puerto Rico, US Virgin Islands, Guam)
  • Alaska and Hawaii

Each site includes precise GPS coordinates (latitude/longitude) for accurate weather alert matching.

Data Processing
How Storm Scout Works
  1. Data Collection: Backend server queries NOAA API every 15 minutes
  2. Alert Matching: Incoming weather alerts are matched to testing centers by geographic location and affected zones
  3. Normalization: Weather alerts from different sources are standardized into a common format
  4. Status Calculation: Site operational status (Open/Closed/At Risk) is automatically determined based on alert severity and type
  5. Dashboard Display: Frontend displays the pre-processed data without heavy client-side computation
